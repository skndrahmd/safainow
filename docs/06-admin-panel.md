# SafaiNow — Admin Panel

## Overview

The Admin Dashboard is a Next.js web application. It is the control centre for the entire SafaiNow platform. Admin has full real-time visibility and full control over every aspect of the system.

## Modules

### 1. Dashboard Home
- Live feed of all active jobs with current status badge
- Jobs completed today
- New customers this week
- New partners this week
- Top performing partners by jobs completed
- Total commission collected vs outstanding (summary cards)

### 2. Booking Management
- Table of all bookings (live and historical) visible from the moment of creation
- Live status updates without page refresh (Supabase Realtime)
- Columns: booking ID, customer, partner, packages, total price, commission, status, created at
- Click any booking to open full detail view:
  - All booking info (customer, partner, packages, services, house address, total price)
  - Commission amount (25% of total)
  - Customer feedback (thumbs up / down)
  - Full timestamp log per stage
  - Travel time, job duration, total time
  - Cancellation info if applicable (who, when, reason)
- Admin can cancel any booking at any stage with a mandatory reason field

### 3. Partner Management
- List of all partners with status badge (active / suspended)
- Create new partner:
  - Full name
  - Phone number
  - CNIC number
  - Profile picture upload
  - CNIC picture upload
  - System auto-generates 6-digit passcode on save
  - Passcode shown once in a modal ("Save this passcode, it will not be shown again")
- Partner detail page:
  - Profile info
  - Job history table
  - Earnings summary
  - Commission owed and paid
  - Thumbs up count, thumbs down count, positive feedback rate %
  - Cancellation rate
- Edit partner profile details
- Reset passcode (new one generated and shown once)
- Suspend / reactivate partner
- Delete partner permanently

### 4. Customer Management
- List of all registered customers
- Customer detail page:
  - Profile info
  - Full booking history
  - Total spent
- Suspend / reactivate customer account

### 5. Individual Services Management (The Basket)
- List of all individual services (name, price, status)
- Add new service (name, price)
- Edit service name or price inline
- Disable service (hides from custom package options on customer app)
- Delete service (auto-removed from all packages and custom package)

### 6. Package Management
- List of all packages with type badge and status
- Create new package:
  - Name
  - Description (shown on customer app detail page)
  - Type: `cleaning` | `standalone` | `custom`
  - Services: multi-select from active individual services basket only
  - Flat price
- Edit package: name, description, price, constituent services
- Enable / disable package (disabled = hidden from customer app)
- Delete package permanently
- Note: custom type package is system-managed, only one should exist

### 7. Commission & Financials
- Commission ledger table per partner:
  - Columns: partner name, job date, job value, commission amount (25%), status (owed / collected), collected date
- Mark commission as collected (per individual job or bulk per partner)
- Summary cards:
  - Total commission owed (all partners)
  - Total commission collected (this month)
  - Total commission collected (all time)
  - Total outstanding
- Export full ledger as CSV

### 8. Reports
- Bookings per day / week / month (chart + table)
- Top partners by jobs completed
- Top customers by total bookings
- Cancellation rate: overall, per partner, per customer
- Revenue and commission summary
- All reports exportable as CSV

## Real-Time Requirements

The following admin views must update in real time using Supabase Realtime subscriptions:
- Dashboard home job feed
- Booking management table (new bookings, status changes)
- Active booking detail page (stage progression, timestamps)

## Partner Passcode System

- 6-digit numeric passcode
- Auto-generated by the system on partner creation
- Stored as **bcrypt hash** in the database (never plain text)
- Shown to admin **once** in a modal on creation
- Admin shares it with partner manually (call or WhatsApp)
- Admin can reset anytime — new passcode generated, shown once
- If partner forgets, admin resets — no recovery flow needed on partner app
